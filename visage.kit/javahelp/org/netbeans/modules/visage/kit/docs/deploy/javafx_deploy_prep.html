Preparing JavaFX Applications for Deployment

See Also 

Using the Project Properties dialog, you can configure your JavaFX application to execute using one of the available execution models: Standard Execution (standalone), Web Start Execution, Run in Browser, Run in Mobile Emulator (Microsoft Windows and Mac platforms only), and Run in TV Emulator (Microsoft Windows only). The Standard Execution model is chosen by default. After you choose the execution model and you build the project, the IDE creates the necessary files and executes the application locally. The files created can then be used to package the application for deployment outside of the IDE.

Alternatively, you can choose the execution model by selecting it from the combo box on the Run toolbar. Open the combo box and select one of the following:

  • <default> for the Standard Execution.
  • Browser for Run in Brower execution model.
  • Mobile for Run in Mobile Emulator execution model.
  • TV for Run in TV Emulator execution model.
  • Web Start for the Web Start Execution.

To run a standalone application using Standard Execution model:

  1. In the Projects window, right-click your project's root node and select Properties.
    Alternatively, you can select the project in the Projects window and select File > Project Properties (JavaFX_project_name) from the main menu.
  2. In Project Properties dialog, select the Run category.
  3. Select the Standard Execution execution model.
  4. Click OK.
  5. In the Projects window, right-click the project's root node and select Run Project.
    The IDE compiles the project and creates the necessary class files in the project's dist folder. If successful, a separate window appears running your application.

Alternatively, you can choose the Standard Execution model by selecting <default> from the combo box on the Run toolbar.

To run a standalone application from the command line:

  1. Navigate to the project's dist folder.
  2. Type the following at the command prompt:
           javafx -jar <application_jar_name>.jar
    or
          javafx -classpath application_jar_name.jar main_JavaFX_class_name

To run an application using the Java Web Start execution model:

  1. In the Projects window, right-click your project's root node and select Properties.
    Alternatively, you can select the project in the Projects window and select File > Project Properties (JavaFX_project_name) from the main menu.
  2. (Optional) If your application requires access to any secured computer resources or secured Java classes and methods, do the following steps:
    1. In Project Properties dialog, select the Application category.
    2. Under the Web Start Specific Properties section, select the Self Signed JAR and the Pack200 Compression check boxes.
  3. In the Categories pane, select the Run category and select the Web Start Execution radio button on the right pane.
  4. Click OK.
  5. In the Projects window, right-click the project's node again and select Run Project.
    The IDE builds the project first, adds the following files to your project's dist folder, and then executes the project.
    • <app-name>.html - sample HTML page with the applet
    • <app-name>_browser.jnlp - JNLP descriptor file for JNLP-based Applet
    • <app-name>.jnlp - JNLP descriptor for direct WebStart execution
    • <app-name>.jar - application JAR file
    • <app-name>.pack.gz - Pack200 compressed application JAR file (if Pack200 option is set)
    • lib folder with user libraries necessary for execution and, optionally, with Pack200-compressed user libraries.

Alternatively, you can choose the Java Web Start execution model by selecting Web Start from the combo box on the Run toolbar.

To run a standalone application with Java Web Start:

Do one of the following after you build the application for Java Web Start execution:
  • Navigate to the project's dist folder and double-click the launch.jnlp file.
  • Navigate to the project's dist folder and type the following at the command prompt:
           javaws <app-name>.jnlp

To run a JavaFX application in a web browser:

  1. In the Projects window, right-click your project's root node and select Properties.
    Alternatively, you can select the project in the Projects window and select File > Project Properties (JavaFX_project_name) from the main menu.
  2. In Project Properties dialog, select the Application category.
  3. Under the Applet Specific Properties section, set the desired width and height to use for your applet. The default is 200 pixels by 200 pixels.
  4. (Optional) Unselect the Draggable Applet checkbox, if desired. It is selected by default.
    This option allows your application to be dragged form the web browser to the desktop, if your browser has the correct version of the Java Runtime Environment (JRE) that supports this feature.
  5. In the Categories pane, select the Run category.
  6. Select the Run in Browser radio button.
  7. Click OK.
  8. In the Projects window, right-click the project's root node again and select Run Project.
    Your system's default web browser appears, respond to the questions in the dialogs that appear, and the application executes.

Alternatively, you can choose the Run in Browser execution model by selecting Browser from the combo box on the Run toolbar.

To run a JavaFX applet outside of the IDE:

  • Navigate to the project's dist folder and double-click the the <projectname>.html file.

To run an application in the Mobile Emulator:

  1. In the Projects window, right-click your project's root node and select Properties.
    Alternatively, you can select the project in the Projects window and select File > Project Properties (JavaFX_project_name) from the main menu.
  2. In Project Properties dialog, select the Run category.
  3. Select the Run in Mobile Emulator radio button.
  4. Click OK.
  5. In the Projects window, right-click the project's root node again and select Run Project.

Alternatively, you can choose the Run in Mobile Emulator execution model by selecting Mobile from the combo box on the Run toolbar.
