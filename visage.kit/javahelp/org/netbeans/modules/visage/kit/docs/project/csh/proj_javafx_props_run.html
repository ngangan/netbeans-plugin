JavaFX Project Properties Dialog Box: Run

See Also 

The Run panel enables you to configure options for executing your JavaFX project locally. You can set the main class and the application execution model to use for deployment. You can also specify a space-separated list of JVM options in this panel.

You open the JavaFX Project Properties dialog box by right-clicking the JavaFX project's root node  and choosing Properties from the pop-up menu.

The application execution models available are the following:

  • Standard Execution. The IDE uses settings that you specify in the project's Properties dialog box to build the project's files. The IDE runs the application from the class files in the project's dist folder.
  • WebStart Execution. If selected, the IDE creates the necessary artifacts to execute your application using Java Web Start. These artifacts can be found in the project's dist folder and can be used for the application's deployment outside of the IDE.
  • Run in Browser. The IDE generates the necessary files, including a <project_name>.HTML file, to enable the application to run in a web browser. The generated files can be found in the project's dist folder. When executed the application is run as an applet in the web browser that the IDE uses for displaying web content, usually the system's default browser.
  • Run in Mobile Emulator. When selected, the IDE creates the necessary .JAD (Java Application Descriptor) files and .JAR (Java Archive) files for your application. It then executes the application using the mobile emulator tool. JAD files describe the MIDlets (Java ME applications) that are distributed as JAR files. This option is currently available on the Microsoft Windows platform only.

Notes:

  • The Browse button for the Main Class field only shows classes in the project's source folders. If you want to specify a class in one of the JAR files on the project's classpath, type the fully qualified class name in the Main Class field.
