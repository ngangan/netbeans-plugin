Profiling Visage Applications

See Also

Profiling is the process of examining an application in order to locate memory or performance-related issues. When profiling a Visage application, you can monitor the Java Virtual Machine (JVM) and obtain data about application performance, including method timing, object allocation and garbage collection. You can use this data to locate potential areas in your code that can be optimized to improve performance. Currently, you can only view raw Java data types and not Visage data types.

When analyzing a Visage application's performance, you can specify to analyze the performance of either the entire application or you can specify the root methods to limit the code that is instrumented and control the classes that are profiled.
