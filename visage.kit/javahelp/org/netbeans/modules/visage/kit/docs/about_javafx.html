Visage support in NetBeans IDE includes the following features:

Visage SDK

The Visage Software Development Kit (SDK) provides the command-line tools and technologies to build and deploy rich internet applications to browsers, desktops, and mobile devices. The Visage SDK includes the following components:

  • Visage Desktop runtime
  • Visage Mobile runtime (Microsoft Windows and Mac only)
  • Visage Mobile Emulator (Microsoft Windows and Mac only)
  • Visage TV Emulator (Microsoft Windows only)
  • Visage API documentation
  • Samples

It is bundled with the Visage plugin for NetBeans and is installed in the visage/visage-sdk directory in your NetBeans installation directory. It is set as the default platform to use for the Visage projects created in the IDE. For more information, see Setting the Target Visage SDK in a Visage Project.

Visage Compiler

The Visage Compiler is also integrated with the Visage plugin and is used to compile Visage source code. Java source code that is included in the Visage projects are compiled with the Java compiler. The results of the compilation are placed into one JAR file, which can be used for deployment or distribution. For more information, see About Building Visage Applications.

Advanced Source Code Editor

The language-aware editor indents and completes your source code written in the Visage Script programming language. The editor also performs semantic syntax highlighting, live parsing of your code, matches words and brackets, marks errors and occurrences, has code completion functionality, and has code folding capability. It has a toolbar that includes controls to enable or disable the preview feature. For more information, see Editing Visage Source Files.

Visage Palette

The Visage Palette provides code snippets that you can use to quickly build your Visage application. The code snippets can be dragged and dropped into the source file that is currently opened in the source editor. For more information, see About the Visage Palette.

Preview Feature

The Visage plugin includes a preview feature that allows you to view the effects of the changes you make to your source code without having to build and run your application each time you make any changes. If there are no compilation errors encountered and the source file represents UI components, Shape, or Frame classes, changes you make in the source editor will be displayed in the canvas pane located just above the source editor pane. For more information, see Using Visage Preview Feature.

Debugger

Currently, the debugger feature included with the Visage plugin supports debugging of both Visage and Java source code that is included in a Visage project. For more information, see Debugging Visage Applications.

Profiler

The Profiler feature is available at the project-level only and not at the source level for Visage projects. For more information, see Profiling Visage Applications.

Package for Deployment

The Project Properties dialog allows you to configure and choose the execution model to use to run your Visage application. The IDE uses this configuration to create the necessary files to package the application for deployment outside of the IDE. For more information, see Preparing Visage Applications for Deployment.
