Generating a UI Stub File for a Visage Graphic

See Also 

When a Visage graphic in FXZ format is included in a NetBeans project, developers can use it to generate a UI Stub file. A UI stub file is a shortcut for loading the graphic in the Visage application. A UI stub file extends FXDNode, loads the graphic, and declares a variable for each graphic object that has an ID value. You can then simply refer to the generated class and variables in your Visage application, because the graphic is loaded in the UI stub.

The UI stub file that is generated has the same name as the Visage graphic plus a UI suffix. For example, if the Visage graphic is named mediaplayer.fxz, the stub file is named mediaplayerUI.fx. The class that is generated also has the name mediaplayerUI.

You do not need to generate a UI Stub file to use an FXZ graphic in a Visage application. You can load Visage graphics in several ways. For examples of loading graphic in Visage applications, see the how-to topic How do I load Production Suite graphics?

Prerequisites

  • Before performing this procedure, you must add the Visage graphic to a NetBeans project. Be sure to identify the Visage graphic with an FXZ or FXD extension.

Procedure

  1. In the NetBeans IDE, open the project for your Visage application.
  2. Locate the FXZ file in the Projects window.
  3. Right-click (or Control-click on the Mac) the FXZ file and select Generate UI stub.
    The file is generated and appears in the Projects window.
