JavaFX Project Properties Dialog Box: Application

See Also 

The Application pane lets you define properties for the presentation of your final application. These properties are used if Java Web Start is enabled for your application and if your application is based on the Swing Application Framework.

You open the JavaFX Project Properties dialog box by right-clicking the JavaFX project's main node and choosing Properties.

The available properties on this page are the following:

Common Application Properties.
  • Name. The name of your application.
  • Vendor. The name of the company that produces the application. The default value is your system userid.
Applet Specific Properties.
  • Width. Specifies the width of the applet, in pixels. The default is 200.
  • Height. The height of the applet in pixels. The default is 200.
  • Draggable Applet. The property to make the applet draggable from your web browser to your desktop. Your web browser needs to have draggable applet support for this feature to work.
Web Start Specific Properties.
  • Codebase. Specifies whether the web start is to be executed locally or deployed using a user defined specification. The default is Local Execution.
  • Codebase URL. A URL representing the location of the application's runtime files. For Local Execution codebase setting, the URL is determined automatically for you.
  • Self Signed JAR . If selected, the application is signed using a certificate that is generated when the project is built. This certificate is not certified by any Certification Authority. Self-signing an application enables the application to do things such as write files to disk and access the network.
  • Pack 200 Compression. If selected, the application files are highly compressed and can be directly deployed, which saves bandwidth and reduces download time.
