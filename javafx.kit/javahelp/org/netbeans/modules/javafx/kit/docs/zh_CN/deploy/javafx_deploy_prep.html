在“项目属性”对话框中，可采用下列可用执行模型之一配置要执行的 JavaFX 应用程序：标准执行（独立）、Web Start 执行、在浏览器中运行，以及在移动仿真器中运行（仅适用于 Microsoft Windows 平台）。缺省情况下，“标准执行”模型处于选中状态。在您选择执行模型和生成项目之后，IDE 会创建必需的文件，并在本地执行应用程序。然后使用所创建的文件打包应用程序，以便在 IDE 外部进行部署。

另外，您还可以从“运行”工具栏的组合框中选择执行模型。打开组合框，然后选择以下选项之一：

  • <缺省>，即选择“标准执行”。
  • 浏览器，即选择“在浏览器中运行”执行模型。
  • 移动，即选择“在移动仿真器中运行”执行模型。
  • Web Start，即选择“Web Start 执行”。

使用“标准执行”模型运行独立应用程序：

  1. 在“项目”窗口中，右键单击项目的根节点，然后选择“属性”。
    另外，您还可以在“项目”窗口中选择项目，从主菜单中选择“文件”>“项目属性”(JavaFX_project_name)。
  2. 在“项目属性”对话框中，选择“运行”类别。
  3. 选择“标准执行”执行模型。
  4. 单击“确定”。
  5. 在“项目”窗口中，右键单击项目的根节点，然后选择“运行项目”。
    IDE 编译项目并在项目的 dist 文件夹中创建必需的类文件。如果成功，则会单独显示一个运行应用程序的窗口。

另外，您还可以通过从“运行”工具栏的组合框中选择 <缺省> 来选择“标准执行”模型。

从命令行运行独立应用程序：

  1. 导航至项目的 dist 文件夹。
  2. 在命令提示符下键入以下命令：
           javafx -jar <application_jar_name>.jar
    或者
          javafx -classpath application_jar_name.jar main_JavaFX_class_name

使用 Java Web Start 执行模型运行应用程序：

  1. 在“项目”窗口中，右键单击项目的根节点，然后选择“属性”。
    另外，您还可以在“项目”窗口中选择项目，从主菜单中选择“文件”>“项目属性”(JavaFX_project_name)。
  2. （可选）如果您的应用程序需要访问任何安全的计算机资源或安全的 Java 类和方法，请执行以下步骤：
    1. 在“项目属性”对话框中，选择“应用程序”类别。
    2. 在“特定于 Web Start 的属性”部分，选中“自签名 JAR”和“Pack200 压缩”复选框。
  3. 在“类别”窗格中，选择“运行”类别，然后选择右窗格上的“Web Start 执行”单选按钮。
  4. 单击“确定”。
  5. 在“项目”窗口中，再次右键单击项目节点，然后选择“运行项目”。
    IDE 会首先生成项目，将以下文件添加到项目的 dist 文件夹中，然后执行项目。
    • <app-name>.html－具有 applet 的样例 HTML 页面
    • <app-name>_browser.jnlp－基于 JNLP 的 Applet 的 JNLP 描述符文件
    • <app-name>.jnlp－直接 WebStart 执行的 JNLP 描述符
    • <app-name>.jar－应用程序 JAR 文件
    • <app-name>.pack.gz－Pack200 压缩的应用程序 JAR 文件（如果设置了 Pack200 选项）
    • lib 文件夹，包含执行所必需的用户库，以及 Pack200 压缩的用户库（可选）。

另外，您还可以通过从“运行”工具栏的组合框中选择 "Web Start" 来选择 "Java Web Start" 执行模型。

使用 Java Web Start 运行独立应用程序：

在为 Java Web Start 执行生成应用程序之后，请执行以下操作之一：
  • 导航到项目的 dist 文件夹，然后双击 launch.jnlp 文件。
  • 导航到项目的 dist 文件夹，然后在命令提示符下键入以下命令：
           javaws <app-name>.jnlp

在 Web 浏览器中运行 JavaFX 应用程序：

  1. 在“项目”窗口中，右键单击项目的根节点，然后选择“属性”。
    另外，您还可以在“项目”窗口中选择项目，从主菜单中选择“文件”>“项目属性”(JavaFX_project_name)。
  2. 在“项目属性”对话框中，选择“应用程序”类别。
  3. 在“特定于 Applet 的属性”部分下，设置 applet 所需使用的宽度和高度。缺省值为 200 像素 x 200 像素。
  4. （可选）如果需要，取消选中“可拖动的 Applet”复选框。缺省情况下，它处于选中状态。
    此选项允许将应用程序从 Web 浏览器拖动到桌面（如果您的浏览器具有支持此功能的正确 Java Runtime Environment (JRE) 版本）。
  5. 在“类别”窗格中，选择“运行”类别。
  6. 选择“在浏览器中运行”单选按钮。
  7. 单击“确定”。
  8. 在“项目”窗口中，再次右键单击项目的根节点，然后选择“运行项目”。
    此时会显示您系统的缺省 Web 浏览器，响应在对话框中出现的问题，然后会执行应用程序。

另外，您还可以通过从“运行”工具栏的组合框中选择“浏览器”来选择“在浏览器中运行”执行模型。

在 IDE 外部运行 JavaFX applet：

  • 导航到项目的 dist 文件夹，然后双击 <projectname>.html 文件。

在移动仿真器中运行应用程序：

  1. 在“项目”窗口中，右键单击项目的根节点，然后选择“属性”。
    另外，您还可以在“项目”窗口中选择项目，从主菜单中选择“文件”>“项目属性”(JavaFX_project_name)。
  2. 在“项目属性”对话框中，选择“运行”类别。
  3. 选择“在移动仿真器中运行”单选按钮。
  4. 单击“确定”。
  5. 在“项目”窗口中，再次右键单击项目的根节点，然后选择“运行项目”。

另外，您还可以通过从“运行”工具栏的组合框中选择“移动”来选择“在移动仿真器中运行”执行模型。
