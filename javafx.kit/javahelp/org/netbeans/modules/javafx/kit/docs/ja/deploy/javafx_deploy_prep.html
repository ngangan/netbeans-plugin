JavaFX アプリケーションを配備するための準備

関連項目 

「プロジェクトプロパティー」ダイアログを使用して、「標準実行」(スタンドアロン)、「Web Start 実行」、「ブラウザで実行」、および「モバイルエミュレータで実行」(Microsoft Windows プラットフォームのみ) の使用可能な実行モデルのいずれかを使用して実行されるように JavaFX アプリケーションを設定できます。デフォルトでは、「標準実行」モデルが選択されます。実行モデルを選択し、プロジェクトを構築すると、IDE で必要なファイルが作成されて、アプリケーションがローカルで実行されます。作成されたファイルを使用して、IDE の外部で配備するアプリケーションをパッケージ化できます。

または、実行ツールバーのコンボボックスから実行モデルを選択することもできます。コンボボックスを開き、次のいずれかを選択します。

  • 「<デフォルト>」(「標準実行」)。
  • 「ブラウザ」(「ブラウザで実行」実行モデル)。
  • 「モバイル」(「モバイルエミュレータで実行」実行モデル)。
  • 「Web Start」(「Web Start 実行」)。

「標準実行」モデルを使用してスタンドアロンアプリケーションを実行する

  1. 「プロジェクト」ウィンドウでプロジェクトのルートノードを右クリックし、「プロパティー」を選択します。
    または、「プロジェクト」ウィンドウでプロジェクトを選択し、メインメニューの「ファイル」>「プロジェクトプロパティー」(JavaFX_project_name) を選択することもできます。
  2. 「プロジェクトプロパティー」ダイアログで「実行」カテゴリを選択します。
  3. 「標準実行」実行モデルを選択します。
  4. 「了解」をクリックします。
  5. 「プロジェクト」ウィンドウでプロジェクトのルートノードを右クリックし、「プロジェクトを実行」を選択します。
    IDE は、プロジェクトをコンパイルし、プロジェクトの dist フォルダに必要なクラスファイルを作成します。成功すると、アプリケーションを実行する別のウィンドウが表示されます。

または、実行ツールバーのコンボボックスから「<デフォルト>」を選択して、「標準実行」モデルを選択することもできます。

スタンドアロンのアプリケーションをコマンド行から実行する

  1. プロジェクトの dist フォルダまで移動します。
  2. コマンドプロンプトで次のように入力します。
           javafx -jar <application_jar_name>.jar
    または
          javafx -classpath application_jar_name.jar main_JavaFX_class_name

「Java Web Start」実行モデルを使用してアプリケーションを実行する

  1. 「プロジェクト」ウィンドウでプロジェクトのルートノードを右クリックし、「プロパティー」を選択します。
    または、「プロジェクト」ウィンドウでプロジェクトを選択し、メインメニューの「ファイル」>「プロジェクトプロパティー」(JavaFX_project_name) を選択することもできます。
  2. (省略可能) セキュリティー保護されたコンピュータリソース、またはセキュリティー保護された Java クラスとメソッドにアプリケーションからアクセスする必要がある場合は、次の手順を実行します。
    1. 「プロジェクトプロパティー」ダイアログで「アプリケーション」カテゴリを選択します。
    2. 「Web Start 固有のプロパティー」セクションで、「自己署名付き JAR」および「Pack200 圧縮」チェックボックスをオンにします。
  3. 「カテゴリ」区画で、「実行」カテゴリを選択し、右側の区画で「Web Start 実行」ラジオボタンを選択します。
  4. 「了解」をクリックします。
  5. 「プロジェクト」ウィンドウでプロジェクトのノードをもう一度右クリックし、「プロジェクトを実行」を選択します。
    IDE は、最初にプロジェクトを構築し、次のファイルを dist フォルダに追加してから、プロジェクトを実行します。
    • <app-name>.html - アプレットを含むサンプルの HTML ページ
    • <app-name>_browser.jnlp - JNLP ベースアプレット用 JNLP 記述子ファイル
    • <app-name>.jnlp - WebStart の直接実行用 JNLP 記述子
    • <app-name>.jar - アプリケーション JAR ファイル
    • <app-name>.pack.gz - Pack200 圧縮アプリケーション JAR ファイル (Pack200 オプションが設定されている場合)
    • 実行に必要なユーザーライブラリ、およびオプションで Pack200 圧縮されたユーザーライブラリを含む lib フォルダ。

または、実行ツールバーのコンボボックスから「Web Start」を選択して「Java Web Start」実行モデルを選択することもできます。

Java Web Start を使用してスタンドアロンアプリケーションを実行する

Java Web Start 実行用にアプリケーションを構築した後に次のいずれかを実行します。
  • プロジェクトの dist フォルダに移動し、launch.jnlp ファイルをダブルクリックします。
  • プロジェクトの dist フォルダに移動し、コマンドプロンプトで次のように入力します。
           javaws <app-name>.jnlp

JavaFX アプリケーションを Web ブラウザで実行する

  1. 「プロジェクト」ウィンドウでプロジェクトのルートノードを右クリックし、「プロパティー」を選択します。
    または、「プロジェクト」ウィンドウでプロジェクトを選択し、メインメニューの「ファイル」>「プロジェクトプロパティー」(JavaFX_project_name) を選択することもできます。
  2. 「プロジェクトプロパティー」ダイアログで「アプリケーション」カテゴリを選択します。
  3. 「アプレット固有のプロパティー」セクションで、アプレットに使用する目的の幅と高さを設定します。デフォルトは 200 ピクセル x 200 ピクセルです。
  4. (省略可能)「ドラッグ可能アプレット」チェックボックスを必要に応じてオフにします。このチェックボックスはデフォルトではオンになっています。
    このオプションを選択すると、この機能をサポートする Java Runtime Environment (JRE) の正しいバージョンが設定されているブラウザでは、アプリケーションを Web ブラウザからデスクトップにドラッグできます。
  5. 「カテゴリ」区画で、「実行」カテゴリを選択します。
  6. 「ブラウザで実行」ラジオボタンを選択します。
  7. 「了解」をクリックします。
  8. 「プロジェクト」ウィンドウで、プロジェクトのルートノードをもう一度右クリックし、「プロジェクトの実行」を選択します。
    システムのデフォルトの Web ブラウザが表示され、表示されたダイアログの質問に回答すると、アプリケーションが実行されます。

または、実行ツールバーのコンボボックスから「ブラウザ」を選択して、「ブラウザで実行」実行モデルを選択することもできます。

JavaFX アプレットを IDE の外部で実行する

  • プロジェクトの dist フォルダに移動して、<projectname>.html ファイルをダブルクリックします。

アプリケーションをモバイルエミュレータで実行する

  1. 「プロジェクト」ウィンドウでプロジェクトのルートノードを右クリックし、「プロパティー」を選択します。
    または、「プロジェクト」ウィンドウでプロジェクトを選択し、メインメニューの「ファイル」>「プロジェクトプロパティー」(JavaFX_project_name) を選択することもできます。
  2. 「プロジェクトプロパティー」ダイアログで「実行」カテゴリを選択します。
  3. 「モバイルエミュレータで実行」ラジオボタンを選択します。
  4. 「了解」をクリックします。
  5. 「プロジェクト」ウィンドウで、プロジェクトのルートノードをもう一度右クリックし、「プロジェクトの実行」を選択します。

または、実行ツールバーのコンボボックスから「モバイル」を選択して、「モバイルエミュレータで実行」実行モデルを選択することもできます。
