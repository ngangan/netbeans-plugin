JavaFX Project Properties Dialog Box: Run

See Also 

The Run page lets you configure options for running your JavaFX project. You can set the main class and the application execution model to use for deployment. You can also specify a space-separated list of VM options in this panel.

You open the JavaFX Project Properties dialog box by right-clicking the JavaFX project's main node  and choosing Properties.

The available application execution models on this page are the following:

  • Standard Execution. The IDE uses settings that you specify in the project's Properties dialog box to build the project's files. The IDE runs the application from the class files in the project's dist folder.
  • WebStart Execution. If selected, the IDE creates the necessary artifacts to make the application deployable with Java Web Start when you build the application. These artifacts can be found in the project's dist folder.
  • Run in Browser. The IDE generates the necessary files, including a <project_name>.HTML file, to make the application run in a web browser. The generated files can be found in the project's dist folder. When executed the application is run as an applet in your system's default web browser.
  • Run in Mobile Emulator. When selected, the IDE creates the necessary .JAD (Java Application Descriptor) files and .JAR (Java Archive) files for your application. It then executes the application using the emulator binary (included with theJavaFX SDK) according to Unified Emulator Interface specification. JAD files describe the MIDlets (Java ME applications) that are distributed as JAR files. This option is currently available on the Microsoft platform only.

Notes:

  • The Browse button for the Main Class field only shows classes in the project's source folders. If you want to specify a class in one of the JAR files on the project's classpath, type the fully qualified class name in the Main Class field.
  • Arguments are passed only to the main class, not to individual files.
