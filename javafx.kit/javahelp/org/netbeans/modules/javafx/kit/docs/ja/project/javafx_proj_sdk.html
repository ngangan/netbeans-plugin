JavaFX プロジェクトにおけるターゲット JavaFX SDK の設定

関連項目 

デフォルトでは、新しく作成された JavaFX プロジェクトは JavaFX Plugin にバンドルされている JavaFX SDK を使用します。JavaFX SDK は、JavaFX アプリケーションのコンパイル、実行、デバッグ、およびプロファイルに使用されるデフォルトプラットフォームです。このプラットフォームは、バンドルされている JavaFX ライブラリと、IDE のインストール時に使用するよう選択した Java SE プラットフォーム (JDK) に基づいています。IDE インストールで使用している JDK バージョンを表示するには、「ヘルプ」>「バージョン情報」を選択し、「詳細」タブをクリックします (MacOS プラットフォームでは、「NetBeans」>「バージョン情報」を選択します)。「Java」フィールドに JDK のバージョンが表示されます。

  • IDE で異なるバージョンの JDK を使用する場合は、コマンド行または IDE-HOME/etc/netbeans.conf ファイル内で --jdkhome <jdk-home-dir> スイッチをオンに指定して IDE を起動します。詳細については、 IDE の起動パラメータ を参照してください。

JavaFX プロジェクトにデフォルトのプラットフォームを設定する

  1. 「プロジェクト」ウィンドウでプロジェクトのルートノードを右クリックし、「プロパティー」を選択します。
    または、Windows プラットフォームの場合、「プロジェクト」ウィンドウでプロジェクトを選択し、メインメニューの「ファイル」>「プロジェクトプロパティー」(JavaFX_project_name) を選択します。
  2. 「プロジェクトプロパティー」ダイアログで、左区画の「ライブラリ」ノードを選択します。
  3. 「JavaFX プラットフォーム」コンボボックスで目的の JavaFX プラットフォームを選択し、「了解」をクリックします。

    JavaFX プロジェクト用のターゲットプラットフォームを切り替えると、処理は次のように変更されます。

    • コード補完の実行時には、新規ターゲットプラットフォームのクラスが優先して使用されます。
    • ターゲットの SDK のソースコードと Javadoc ドキュメントがある場合は、それらが表示されます。
    • アプリケーションのコンパイルおよび実行に、ターゲット JDK の実行可能ファイル (javafxc および javafxrt) が使用されます。
    • ターゲット SDK のライブラリを使用してソースコードをコンパイルします。

追加の JavaFX プラットフォームを IDE に登録する

  1. IDE のメインメニューから「ツール」>「Java プラットフォーム」を選択します。
  2. 「Java プラットフォームマネージャー」で「プラットフォームを追加」ボタンをクリックします。
  3. 「Java プラットフォームを追加」ウィザードで、JavaFX プラットフォームの種類を選択し、「次へ」をクリックします。
  4. 新しいプラットフォーム設定に割り当てるプラットフォーム名を指定します。
  5. 新しい JavaFX プラットフォームで使用する必要な JavaFX ライブラリを保存するフォルダを指定します。「参照」ボタンをクリックして、使用する JavaFX SDK インストールのフォルダに移動します。選択したプラットフォームがサポートされない場合は、警告が表示されます。このプラットフォームで処理を続行しないでください。
  6. 「完了」をクリックします。
    新しいプラットフォームが、使用可能な JavaFX プラットフォームのリストに追加されます。
  7. 「閉じる」をクリックして、「Java プラットフォームマネージャー」ダイアログを閉じます。
