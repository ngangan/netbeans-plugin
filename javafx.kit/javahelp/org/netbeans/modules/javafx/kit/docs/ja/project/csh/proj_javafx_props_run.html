JavaFX の「プロジェクトプロパティー」ダイアログ:「実行」

関連項目 

「実行」パネルでは、JavaFX プロジェクトをローカルに実行するためのオプションを設定できます。配備に使用する主クラスとアプリケーション実行モデルを設定できます。このパネルでは、JVM オプションの、スペースで区切ったリストを指定することもできます。

JavaFX の「プロジェクトプロパティー」ダイアログを開くには、JavaFX プロジェクトのルートノードを右クリックし、ポップアップメニューから「プロパティー」を選択します。

使用可能なアプリケーション実行モデルは次のとおりです。

  • 標準実行。IDE は、プロジェクトの「プロパティー」ダイアログで指定された設定を使用してプロジェクトのファイルを構築します。IDE は、プロジェクトの dist フォルダ内のクラスファイルからアプリケーションを実行します。
  • WebStart 実行。選択した場合、IDE は Java Web Start を使用してアプリケーションを実行するために必要な成果物を作成します。成果物は、プロジェクトの dist フォルダにあり、IDE の外部にアプリケーションを配備するために使用できます。
  • ブラウザで実行。IDE は、アプリケーションを Web ブラウザで実行するために必要な <project_name>.HTML ファイルなどのファイルを生成します。生成されたファイルは、プロジェクトの dist フォルダに保存されています。実行すると、アプリケーションは、IDE が Web コンテンツの表示に使用する Web ブラウザ (通常はシステムのデフォルトブラウザ) でアプレットとして実行されます。
  • モバイルエミュレータで実行。選択すると、IDE は必要な Java Application Descriptor (.JAD) ファイルと Java Archive (.JAR) ファイルを作成します。この場合は、モバイルエミュレータツールを使用してアプリケーションが実行されます。JAD ファイルには、JAR ファイルとして配布される MIDlet (Java ME アプリケーション) が記述されます。このオプションは現在 Microsoft Windows プラットフォームでのみ選択できます。

注

  • 「主クラス」フィールド上にある「参照」ボタンをクリックすると、プロジェクトのソースフォルダ内にあるクラスだけが表示されます。プロジェクトのクラスパス上にある JAR ファイル内のクラスを指定する場合は、「主クラス」フィールドに目的のクラスの完全修飾名を指定します。
