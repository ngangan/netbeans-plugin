Move a Class

See Also

You can move a class to another package and change all references to the class in its new location..

To move a class and change the code that references that class:

  1. In the Projects window or Source editor right-click the class and choose Refactor > Move from the contextual menu.
  2. In the Move Class dialog box, select the package from the To Package combo box or type the fully-qualified package name.
  3. If you click the Refactor button, the IDE instantly applies all changes. If you click the Preview button, the Refactoring window displays all lines of code that will be changed. Review the changes and clear the checkbox for any code that you do not want to change. Click the Do Refactoring button to apply the changes.
  • You can also initiate the moving of a class by dragging the class's node in the Project's window to another package or by cutting and pasting the class's node.

To move a class without doing refactoring:

  1. In the Projects window, manually move the class to another package.
    You can cut and paste the class, or you can use drag and drop.
  2. In the Move Class dialog box, select the Move Without Refactoring checkbox.
  3. Click Next.
  • You should always perform a clean build after completing any refactoring commands. To perform a clean build, right-click the project's node in the Projects window and choose Clean and Build Project.
