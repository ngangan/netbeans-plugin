通过将一组类文件添加到 JavaFX 项目的类路径中，可以通知 IDE 在编译和执行期间此项目应该能够访问哪些类。IDE 还使用类路径设置来启用代码完成、自动突出显示编译错误以及重构。

  • 源根目录只能位于单个项目中，不能与其他项目共享，无论这些项目打开与否。如果需要在几个项目中使用某个库，请创建一个特殊项目来存储它。

类路径和 JavaFX 项目

IDE 可分别维护用于编译和运行 JavaFX 项目的类路径。IDE 自动将项目编译类路径中的所有内容添加到项目的运行时类路径中。

  • 每当生成指定了主类的 JavaFX 项目时，IDE 都会自动将此项目类路径中的任何 JAR 文件复制到 dist/lib 文件夹中。IDE 还会将每个 JAR 文件添加到应用程序 JAR 的 manifest.mf 文件的 Class-Path 元素中。这会简化在 IDE 外部运行应用程序的过程。

编辑 JavaFX 项目的类路径：

  1. 展开项目节点，右键单击“库”节点，然后选择“属性”。
    另外，您还可以在“项目”窗口中选择项目，从主菜单中选择“文件”>“项目属性”(<JavaFX_project_name>)，然后从“项目属性”对话框中选择“库”类别。
  2. 在“项目属性”对话框中，通过单击相应的“添加”按钮，将必要的元素添加到项目的编译类路径中。您可以添加以下任意内容：
    • 项目。其他项目生成的 JAR 文件或 WAR 文件以及关联的源文件和 Javadoc 文档。
    • 库。包含已编译类的 JAR 文件或文件夹的集合，这些文件或文件夹可以有选择地包含关联的源文件和 Javadoc 文档。
    • JAR/文件夹。包含编译的类的 JAR 文件或文件夹。
  3. （可选）在生成当前项目时，如果不想重新生成类路径上的所有项目，请单击“在类路径上生成项目”复选框。缺省情况下，在生成当前项目时，将重新生成类路径上和 Web 应用程序中的所有项目，以及“包装”页中列出的项目。
  4. （可选）单击“上移”和“下移”按钮来更改类路径优先级。
  5. （可选）单击“运行”、“编译测试”或“运行测试”标签对这些类路径进行任何更改。
  • 也可以使用以下方法将相关项目和其他必需资源添加到项目中：只需在“项目”窗口中右键单击“库”节点，然后从上下文菜单中选择相应项（“添加项目”、“添加库”或“添加 JAR/文件夹”）。
